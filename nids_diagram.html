Entry Point

The application's starting point. It instantiates the IntrusionDetectionSystem, loads the pre-trained Random Forest model from disk, and calls ids.start() to begin the monitoring loop. The network interface name is hardcoded here and must match your machine.

Responsibilities

app/ids.py
IDS Orchestrator
▾

The central coordinator of the entire system. It wires together all 4 components and runs the main event loop. On each iteration it: pulls a packet from the queue → updates the flow → checks for expired flows → runs detection → fires alerts. Handles graceful SIGINT shutdown (Ctrl+C).

Main Loop Logic
1. Get packet from queue (1s timeout)
2. Update flow via FlowManager
3. Check expired flows
4. Detect threats per flow
5. Log alerts if found
Wires Together
PacketCapture FlowManager DetectionEngine AlertSystem
Signal Handling
Catches SIGINT → stops sniffer thread → exits cleanly
Error Handling
try/except wraps entire loop body; runtime errors are printed but don't crash the system
captures packets

app/packet_capture.py
Packet Sniffer
▾

Uses Scapy's sniff() to capture raw packets from the NIC in a background thread. Only TCP/IP packets are accepted. Each qualifying packet is pushed into a queue.Queue, decoupling capture from processing so no packets are dropped while detection is running.

Technology
Scapy sniff() · Python threading · queue.Queue
Filter
Only passes packets containing IP layer TCP layer
Threading Model
Capture runs in a daemon thread. Main thread consumes from queue with 1s timeout.
Stop Mechanism
threading.Event signals Scapy's stop_filter to halt sniffing gracefully
raw packets → flows

app/flow_manager.py
Flow Manager
▾

Groups individual packets into bidirectional flows using a 5-tuple key. After 2 seconds of inactivity, a flow expires and is finalized into a 17-feature statistical vector ready for detection.

Flow Key (5-tuple)
srcIP · dstIP · srcPort · dstPort · Protocol(TCP)
Timeout
2 seconds of inactivity → flow finalized
Extracted Features (17)
Destination Port
Flow Duration
Total Fwd Packets
Total Bwd Packets
Fwd Bytes
Bwd Bytes
Flow Bytes/s
Flow Packets/s
SYN Flag Count
ACK Flag Count
FIN Flag Count
Packet Length Mean
Packet Length Std
Idle Mean
Active Mean
Source IP
Destination IP
feature vector

app/detection_engine.py
Detection Engine
▾

The brain of the system. Runs two independent detection methods on every flow. Both can trigger simultaneously. Returns a list of threat objects, each with type, attack name, and confidence score.

⚡ RULE-BASED
Port Scan — ≥10 unique destination ports from same source within 5 seconds → confidence 0.95
SYN Flood — SYN count >20 AND ACK count = 0 → confidence 0.90
🤖 ML-BASED
Random Forest trained on CICIDS2017 dataset (~2.8M samples)
15-feature vector → classifies as BENIGN, DoS, PortScan, Web Attack, or Infiltration
Non-BENIGN predictions added as threat with confidence 1.0
threat objects
🚨
app/alert_system.py
Alert System
▾

Receives threat objects and flow features, builds a structured alert record, stores it in memory, and persists the full alert list to JSON on every write. Also loads existing alerts on startup so history is preserved across runs.

Alert JSON Schema
{
  "timestamp": "2025-03-05 14:32:45",
  "attack_type": "PortScan",
  "threat_category": "rule_based",
  "source_ip": "192.168.1.100",
  "destination_ip": "172.16.0.1",
  "destination_port": 22,
  "confidence": 0.95
}
Storage
logs/ids_alerts.json — full rewrite on every alert
Access Methods
get_recent_alerts(limit) · get_total_alerts()
